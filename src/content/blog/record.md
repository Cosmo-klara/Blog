## 一些学习记录

### Embedding

**Embedding 就是让计算机能够"理解"离散数据的含义，并用数学向量的形式表达这种理解。**

#### 为什么需要 Embedding？

原始表示的局限性

```python
# One-hot 编码（传统方法）
"猫" = [1, 0, 0, 0, 0]
"狗" = [0, 1, 0, 0, 0] 
"汽车" = [0, 0, 1, 0, 0]

# 问题：
# 1. 维度灾难（词汇表很大时）
# 2. 所有词之间的距离相等
# 3. 无法表达语义关系
```

解决方案：Embedding

```python
# Embedding 表示
"猫" = [0.8, 0.1, 0.9]
"狗" = [0.7, 0.2, 0.8]      # 与猫相似
"汽车" = [0.1, 0.9, 0.2]    # 与猫狗差异大
"老虎" = [0.75, 0.15, 0.85] # 与猫更相似
```

#### Embedding 的数学特性

语义关系保持：

```python
# 例如：
"国王" → [0.2, 0.8, -0.3, 0.5]
"皇后" → [0.3, 0.7, -0.2, 0.6]
"男人" → [0.1, 0.9, -0.4, 0.4]
"女人" → [0.2, 0.8, -0.3, 0.5]

# 向量("国王") - 向量("男人") + 向量("女人") ≈ 向量("皇后")
```
